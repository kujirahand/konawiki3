# KonaNotation (Kona記法) の書き方

Konawiki3独自のマークアップ言語「KonaNotation (Kona記法)」の書き方を説明します。
KonaNotationは、Markdown (マークダウン) 記法と同様に、プレーンテキストで記述することができます。

このページでは、`kona3engine/kona3parser.inc.php` で処理される主な記法をまとめます。

## 基本

KonaNotationは、行頭の記号で見出し、リスト、表、コード、プラグインなどを判定します。
何も記号を付けない行は通常の段落として扱われます。

```text
これは通常の段落です。
続きの文章を書けます。

空行を入れると、次の段落になります。
```

通常の文章中では、HTMLタグに使われる `<`、`>`、`&`、`"` は自動的にエスケープされます。

## 見出し

見出しは、行頭に `*` を付けて書きます。
`*` の数が見出しレベルになります。

```text
* 見出し1
** 見出し2
*** 見出し3
**** 見出し4
```

日本語向けの記号も使えます。

```text
■ 大見出し
● 中見出し
▲ 小見出し
▼ さらに小さい見出し
```

次の記号も同じレベルの見出しとして扱われます。

| 記号 | レベル |
| ---- | ------ |
| `■` / `□` | 見出し1 |
| `●` / `○` | 見出し2 |
| `▲` / `△` | 見出し3 |
| `▼` / `▽` | 見出し4 |

## 箇条書きリスト

行頭に `-` または `・` を付けると、箇条書きリストになります。

```text
- 項目1
- 項目2
- 項目3
```

```text
・項目1
・項目2
・項目3
```

`-` を重ねると、入れ子のリストになります。

```text
- 親項目
-- 子項目
--- 孫項目
- 次の親項目
```

## 番号付きリスト

行頭に `+` を付けると、番号付きリストになります。

```text
+ 項目1
+ 項目2
++ 子項目
+ 項目3
```

全角の `＋` も使用できます。

## 表

行頭に `|` を付けると、表になります。
セルは `|` で区切ります。

```text
| 名前 | 種類 | 説明 |
| FrontPage | ページ | トップページ |
| Help | ページ | ヘルプ |
```

連続する表の行は1つの表として出力されます。
各セル内では、太字やリンクなどのインライン記法も使えます。

## 水平線

`--` のように、2個以上の `-` だけを書いた行は水平線になります。

```text
--
```

```text
---
```

## コード

行頭に半角スペースまたはタブを入れると、コード行として扱われます。

```text
 通常の文章ではなくコードとして表示されます。
 echo "hello";
```

連続するコード行は、まとめてコードブロックとして表示されます。

## コードブロック

複数行のコードは、`{{{` と `}}}` で囲みます。

```text
{{{
function hello() {
    echo "hello";
}
}}}
```

開始側の `{` を増やした場合は、同じ数の `}` が終了記号になります。
これにより、本文中に `}}}` を含む内容も書きやすくなります。

```text
{{{{
本文中に }}} を含めたい場合
}}}}
```

## ブロックプラグイン

行頭に `#プラグイン名` を書くと、ブロックプラグインを呼び出します。

```text
#comment
```

引数は `()` の中にカンマ区切りで書きます。

```text
#plugin(arg1,arg2,arg3)
```

全角の `♪` もプラグイン呼び出し記号として扱われます。

```text
♪plugin(arg1,arg2)
```

全角チルダ `～` で引数を区切る形式も使えます。

```text
#plugin～arg1～arg2～arg3
```

### 本文付きプラグイン

コードブロックの中身が `#プラグイン名(...)` で始まる場合は、本文付きプラグインとして処理されます。
ブロック本文は、最後の引数としてプラグインへ渡されます。

```text
{{{#code(js)
console.log("hello");
}}}
```

## 引用

行頭に `>` または全角の `＞` を付けると、引用風のブロックになります。

```text
> 引用した文章です。
> 複数行の引用も書けます。
```

連続する引用行は、まとめて引用ブロックとして表示されます。

## リンク

### Wikiページへのリンク

`[[ページ名]]` でWikiページへのリンクになります。

```text
[[FrontPage]]
```

表示名を変える場合は、`[[表示名:ページ名]]` と書きます。

```text
[[トップページ:FrontPage]]
```

`[[[ページ名]]]` という形式も、ページリンクとして扱われます。

```text
[[[FrontPage]]]
```

### URLへのリンク

本文中に `http://`、`https://`、`ftp://` で始まるURLを書くと、自動的にリンクになります。

```text
https://example.com/
```

表示名を付けたURLリンクは、`[[表示名:https://example.com/]]` の形式で書けます。

```text
[[公式サイト:https://example.com/]]
```

`mailto:` で始まるメールアドレスも自動リンクになります。

```text
mailto:info@example.com
```

## インライン装飾

文章中で使える装飾記法です。

| 記法 | 表示 |
| ---- | ---- |
| `''太字''` | 強調表示 |
| <code>``強調``</code> | 別スタイルの強調表示 |
| `%%赤文字%%` | 赤文字 |

例:

```text
これは ''太字'' です。
これは ``別スタイルの強調`` です。
これは %%赤文字%% です。
```

## インラインプラグイン

文章中では、`&プラグイン名(...);` の形式でインラインプラグインを呼び出せます。

```text
本文の途中に &plugin(arg1,arg2); を埋め込みます。
```

プラグインが存在しない場合や無効化されている場合は、プラグインとして実行されません。

## 改行

段落内で明示的に改行したい場合は、行末に `~` を付けます。

```text
1行目~
2行目
```

設定 `para_enabled_br` が有効な場合は、段落内の通常の改行も `<br/>` として扱われます。

## 差分表示用の記法

`>>>[+]` または `>>>[-]` で始まる行は、差分表示用の行として扱われます。
通常のページ作成ではあまり使いませんが、内部的な差分表示で利用されます。

```text
>>>[+] 追加された行
>>>[-] 削除された行
```

## まとめ例

```text
■ KonaNotationの例

これは通常の段落です。
''強調'' や %%赤文字%%、[[FrontPage]] へのリンクを書けます。

● リスト

- 項目1
-- 子項目
- 項目2

● 表

| 名前 | 説明 |
| FrontPage | トップページ |
| Help | ヘルプ |

● コード

{{{
echo "hello";
}}}

● プラグイン

#comment
```
