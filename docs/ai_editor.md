# KonaWiki3 AI執筆支援機能

KonaWiki3の編集画面では、AIを使って文章の校正、要約、翻訳、言い換え、続きを作文する補助機能を利用できます。
AIは自動的に本文を書き換えるのではなく、ユーザーが明示的にボタンやショートカットキーを実行したときだけ動作します。

## 利用できる機能

- AI支援: ひな形を選び、本文や選択範囲を使ってAIに依頼します。
- AIチェック: 誤字脱字チェックのひな形を選び、本文の修正候補を表示します。
- 文章補完: カーソル位置までの文章をもとに、続きを1文だけ生成してカーソル位置へ挿入します。
- 誤字脱字修正ショートカット: 既存のAIチェックをショートカットキーで実行します。

## 初期設定

管理者でログインし、設定画面の「AI」セクションを設定します。
編集画面のAIパネルにある「AI設定を編集」リンクから、設定画面のAIセクションを新しいウィンドウで開けます。

主な設定項目は次の通りです。

- AI Provider: `OpenAI` または `OpenRouter` を選びます。
- OpenAI API Key: OpenAIを使う場合に指定します。
- OpenAI API Model: OpenAIで使うモデルを選びます。
- OpenRouter API Key: OpenRouterを使う場合に指定します。
- OpenRouter Model: OpenRouterで使うモデル名を指定します。
- OpenAI Basic Instruction: AIに渡す基本指示です。
- ショートカットキー > 文章補完: 既定値は `Ctrl+Shift+H` です。
- ショートカットキー > 誤字脱字修正: 既定値は `Ctrl+Shift+J` です。
- ショートカットキー > USER_PROMPT1: 既定値は `Ctrl+Alt+1` です。
- ショートカットキー > USER_PROMPT2: 既定値は `Ctrl+Alt+2` です。

APIキー欄には実際のキーを入力できます。
環境変数を使う場合は、`@ENV:OPENAI_API_KEY` や `@ENV:OPENROUTER_API_KEY` のように指定します。

OpenAIを使う場合、モデル名は設定画面の選択肢から選びます。
OpenRouterを使う場合、利用したいモデル名を `openrouter_model` に直接入力します。
編集画面にはモデル入力欄を表示しません。モデルの変更は設定画面で行います。

## 編集画面での使い方

編集画面で「AI支援」ボタンを押すと、AIパネルが開きます。
AIパネルには、ひな形の選択、ひな形編集リンク、AI設定編集リンク、ヘルプリンク、入力欄、実行ボタン、結果表示欄があります。

1. 「AI支援」を押します。
2. ひな形を選びます。
3. 必要に応じて入力欄の指示を編集します。
4. 「AIに依頼」を押します。
5. 結果欄に表示された内容を確認し、必要に応じて本文に追加、コピー、置換します。

通常のテキスト結果には「Add」と「Copy」ボタンが表示されます。
「Add」を押すと、結果を現在のカーソル位置へ挿入します。

誤字脱字チェックなどのJSON形式の結果では、修正候補ごとに次の操作ができます。

- 置換: 本文中の該当箇所を修正候補で置き換えます。
- 探す: 本文中の該当箇所を選択します。
- Copy: 修正候補をコピーします。
- 取り消す: その候補を結果欄から消します。

## ショートカットキー

設定画面のAIセクションでショートカットキーを変更できます。
既定値は次の通りです。

- `Ctrl+Shift+H`: 文章補完を実行します。
- `Ctrl+Shift+J`: 誤字脱字修正を実行します。
- `Ctrl+Alt+1`: `# USER_PROMPT1` のひな形を実行します。
- `Ctrl+Alt+2`: `# USER_PROMPT2` のひな形を実行します。

ショートカットキーは設定で変更できますが、ブラウザやOSが予約しているキーはページ側で捕捉できない場合があります。
例えば `Ctrl+N` や `Ctrl+J` は多くのブラウザで新規ウィンドウやダウンロード表示に割り当てられています。

文章補完は、カーソル位置より前の文章をAIへ渡し、続きを1文だけ生成します。
生成された文章は、AI結果欄ではなく本文のカーソル位置へ直接挿入されます。

誤字脱字修正は、AIチェックのひな形を使って本文を確認し、結果欄に修正候補を表示します。
USER_PROMPT1とUSER_PROMPT2は、ユーザーが自由に内容を決めるためのショートカット用ひな形です。
`ai_prompt.md` にある `# USER_PROMPT1` または `# USER_PROMPT2` の内容を編集すると、それぞれのショートカットから実行できます。

## ひな形の編集

AIパネルの「ひな形を編集」リンクから、AI用のひな形ページを編集できます。
初回は `data/ai_prompt.md` が作成されます。
このファイルが存在する場合は、ユーザー定義のひな形として読み込まれます。
さらに、言語別の標準ひな形として `kona3engine/lang/ja-ai_prompt.md` なども読み込まれます。
新規作成される `ai_prompt.md` には、最初から `# USER_PROMPT1` と `# USER_PROMPT2` が含まれます。

ひな形は次の形式で記述します。

```text
# ひな形名
### 指示:
ここにAIへの指示を書きます。
### 入力:
__TEXT__

-----
```

`# ` で始まる行が、編集画面のひな形選択リストに表示されます。
ひな形の区切りには `-----` を使います。

利用できる置換文字列は次の通りです。

- `__TEXT__`: 編集中の本文全体に置き換わります。
- `__TEXT_SELECTED__`: 編集画面で選択している範囲に置き換わります。

誤字脱字チェックや置換候補を出したい場合は、AIにJSON形式で返すよう指示します。
結果が `ng`、`ok`、`desc` を持つJSON配列の場合、KonaWiki3は置換候補として表示します。

```json
[
  {
    "ng": "修正前の文章",
    "ok": "修正後の文章",
    "desc": "修正理由"
  }
]
```

## AIヘルプリンク

AIパネルの「？」リンクは、AI機能のヘルプページを新しいウィンドウで開きます。
リンク先は次のページです。

```text
https://kujirahand.com/konawiki3/index.php?AI
```

このページには、本ドキュメントの内容を掲載すると、編集画面から設定方法や使い方を確認しやすくなります。

## トラブルシューティング

AI支援ボタンが表示されない場合は、設定画面のAIセクションでプロバイダーとAPIキーを確認してください。
OpenAIのAPIキーが未設定でも、OpenRouterのAPIキーが設定されていればAI機能は表示されます。

「AI API Key is not set.」と表示される場合は、選択したプロバイダーのAPIキーが設定されていません。
環境変数を使う場合でも、設定欄には `@ENV:...` を指定してください。

結果が期待した形式で表示されない場合は、ひな形の指示を確認してください。
置換候補として表示したい場合は、JSON配列をコードブロックで返すようAIに指示します。
