# マークアップモード自動判定の仕様 (Markup Detection Specification)

KonaWiki3 は、ページを表示 (show) および編集 (edit) する際に、どのマークアップ記法（Markdown または KonaNotation）を使用するかを以下のルールと優先順位に従って判定します。

---

## 判定の優先順位 (Resolution Order)

ページ解決およびレンダリング時には、次の優先順位でマークアップモードが決定されます。

1. **ページのメタデータ設定 (`mode`)**
   - ページに対応するメタデータファイル (`data/.meta/{ページ名}.json`) を読み込みます。
   - JSON 内の `mode` キーの値が `Markdown` または `KonaNotation` である場合、これが最優先されます。
   - **これにより、実際のファイル拡張子が `.txt` であっても、メタデータで `Markdown` が指定されていれば、そのページは Markdown 記法として解釈・レンダリングされます。**

2. **ディスク上の物理ファイルの拡張子**
   - メタデータに `mode` が指定されていない場合、該当するファイルの拡張子を確認します。
   - `.md` の場合は `Markdown` モードとして解釈されます。
   - `.txt` の場合は `KonaNotation` モードとして解釈されます。
   - `.txt` と `.md` の両方がディスク上に存在する場合は、後述のシステムデフォルト設定の拡張子が優先されます。

3. **システム設定のデフォルト値 (`def_text_ext`)**
   - メタデータがなく、ディスク上にもファイルが存在しない新規ページ作成などの場合、システム設定 `def_text_ext` (デフォルト: `txt`) の設定値が適用されます。
   - `def_text_ext` が `txt` の場合は `KonaNotation` 用のファイル (`.txt`) が、`md` の場合は `Markdown` 用のファイル (`.md`) が作成されます。

---

## 編集画面でのマークアップモード切り替え

編集 (edit) 画面において、ユーザーはフォーマットを自由に変更できます。

- 編集フォームの下部（自動保存オプションの右側）に **Format** 切り替えセレクトボックスが表示されます。
- フォーマットを切り替えて保存（通常保存、一時保存、または自動保存）を行うと、サーバー側でページのメタデータファイル (`data/.meta/{ページ名}.json`) が生成・更新され、`mode` キーに選択された形式 (`Markdown` または `KonaNotation`) が書き込まれます。
- テキストエリアを空にしてページを削除した場合、対応するメタデータファイルも自動的に削除されます。
- **既存ページの拡張子維持ポリシー**:
  - 新規ページ（まだファイルが存在しない状態）の作成時は、選択した `Format` に応じてファイルの拡張子（`.md` または `.txt`）が決定されます。
  - すでに物理ファイルが存在する既存ページの場合は、混乱やリンク切れを避けるため、**ファイルの拡張子は変更されません**。この場合、Format の変更はメタデータファイル内の `mode` キーの更新のみとなり、実際の物理ファイルの拡張子は元のまま維持されます。
